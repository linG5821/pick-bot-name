## Pattern Analysis

### English Patterns

1. **Single Word Names** (30%+)
   - Simple, memorable: Fin, Ada, Drift, Snyk, Mio
   - Often 1-2 syllables
   - Easy to pronounce and type
   - Pattern: Branded, unique names

2. **Compound Words** (40%+)
   - Format: [Function] + Bot/bot
   - Examples: Slackbot, Geekbot, Workbot, Helpbot, Supportbot
   - Clear purpose indication
   - Pattern: descriptive + "bot"

3. **Action/Role Names** (20%)
   - Bulldozer, Resolver, Copilot
   - Implies function through metaphor
   - Pattern: verb/noun indicating action

4. **Prefix Patterns** (10%)
   - Format: [Prefix] + [Core]
   - Examples: Dependabot (Depend + a + bot), Probot (Pro + bot)
   - Pattern: function + bot with connector

**Key Insights:**
- Length: 1-3 words max (3-12 characters ideal)
- Avoid: "AI", "Smart", "Intelligent" prefixes (overused)
- Prefer: Unique brand names OR clear [Function]bot format
- Most successful: Single memorable word (Fin, Ada, Drift)

### Chinese Patterns (中文模式)

1. **小X Pattern** (35%+)
   - 小钉, 小飞, 小智, 小Q, 小爱, 小冰, 小度, 小艺, 小客
   - "小" = little/small (cute, friendly)
   - Usually 2 characters total
   - Pattern: 小 + [single character identifier]

2. **X助手 Pattern** (25%)
   - 审批助手, 会议助手, 文档助手, 协作助手, 客服小助手
   - "助手" = assistant
   - Usually 3-5 characters
   - Pattern: [function] + 助手

3. **X君 Pattern** (15%)
   - 考勤君, 销售君, 测试君, 法务君
   - "君" = respectful suffix, professional
   - Usually 3 characters
   - Pattern: [function] + 君

4. **小X手 Pattern** (10%)
   - 小助手, 小管家, 报销小助手, 客服小助手
   - Combination of "小" + function
   - Usually 3-5 characters
   - Pattern: 小 + [function word]

5. **X机器人 Pattern** (5%)
   - 智能机器人, 语雀机器人, 飞书文档机器人, 运维机器人
   - "机器人" = robot (formal, explicit)
   - Usually 4-6 characters
   - Pattern: [function] + 机器人

6. **Direct Function Names** (10%)
   - 智能客服, 在线客服, 智能回复
   - No cute suffix, straightforward
   - Pattern: [adjective] + [function]

**Key Insights:**
- Ideal Length: **2-3 characters** (most common: 小X, X君, X助手)
- Maximum: 4 characters for professional contexts
- Avoid: 
  - 5+ character names (too long, formal)
  - "的" constructions (可爱的小猫助手 - 7 chars, BAD)
  - "机器人" suffix (adds 3 chars, makes names too long)
  
**Most Natural Patterns:**
1. 2-char: 小钉, 小飞, 小智 (friendly, personal)
2. 3-char: 考勤君, 销售君 (professional, role-specific)
3. 3-char: X助手 (客服助手, but avoid "小助手" patterns that become 4+ chars)

**Character Recommendations:**
- Use reduplication for cute styles: 喵喵, 团团, 软软
- Use 君 suffix for professional contexts: 算法君, 代码君
- Use 小X for friendly/approachable: 小智, 小管家
- Avoid "助手" in cute styles (too formal)

## Recommendations for Bot Name Generator Rules

### English Professional Rules
- Focus on single-word brand names (6-10 chars)
- [Function]bot compounds (max 12 chars)
- Avoid "AI", "Smart", "Intelligent" prefixes
- Examples to generate:
  - Taskbot, Flowbot, Syncbot, Alertbot, Codebot
  - Keeper, Scout, Guide, Herald, Proxy
  - Nexus, Vertex, Core, Pulse, Link

### Chinese Professional Rules ⚠️ CRITICAL
- **Primary pattern: 2-3 characters ONLY**
- Small X (小X): 小智, 小助, 小管, 小秘
- Role + 君: 助理君, 文档君, 代码君, 测试君
- Function names: 智能助理 (4 chars max for formal)
- **NEVER generate:**
  - 5+ character names
  - Names with "的" 
  - "可爱的X助手" style (too long, childish)
  - "智能机器人助手" style (redundant, too long)

### Pattern Priority
**English:** Single word > [Function]bot > Metaphor names
**Chinese:** 小X > X君 > Function names (2-3 chars only)
