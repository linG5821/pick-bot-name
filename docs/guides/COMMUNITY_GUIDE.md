# 🌟 Community Guide

Welcome to the Bot Name Generator community! This guide will help you understand how to participate, contribute, and engage with the project.

English | [简体中文](COMMUNITY_GUIDE_zh.md)

## 🎯 Ways to Contribute

### 1. 🎲 Submit Generation Rules

Generation rules are the core of this project. Share your creativity!

**What makes a great rule?**
- Unique and creative combinations
- Culturally appropriate content
- Well-balanced weights
- Clear documentation
- Quality examples

**Process:**
1. Read the [Rule Guide](RULE_GUIDE.md)
2. Create your rule JSON file
3. Test with `npm run validate-rules`
4. Submit a Pull Request
5. Respond to feedback

**Recognition:**
- Your name in the rule file
- Listed in contributors
- Featured in release notes

### 2. 🐛 Report Bugs

Found a bug? Help us fix it!

**Before reporting:**
- Search existing issues
- Check if it's already fixed
- Gather reproduction steps

**Report format:**
- Use the bug report template
- Include screenshots
- Describe expected vs actual behavior
- Provide environment details

### 3. ✨ Suggest Features

Have an idea? We'd love to hear it!

**Good feature requests:**
- Solve a real problem
- Align with project goals
- Include use cases
- Consider implementation

**Process:**
1. Open a feature request issue
2. Describe the problem
3. Propose a solution
4. Discuss with community

### 4. 📚 Improve Documentation

Documentation is always welcome!

**Areas to improve:**
- Tutorials and guides
- API documentation
- Code comments
- Translation fixes
- Examples

### 5. 🌍 Add Translations

Help make the project accessible worldwide!

**Translation needs:**
- UI text (zh, en, ja)
- Documentation
- Rule descriptions
- Error messages

### 6. 💻 Code Contributions

Ready to dive into the code?

**Good first issues:**
- Look for `good-first-issue` label
- UI improvements
- Bug fixes
- Test coverage

**Development setup:**
```bash
npm install
npm run dev
npm run validate-rules
npm run build
```

## ❓ FAQ

**Q: I'm new to open source. Where do I start?**
A: Start with documentation or submit a simple rule. Check `good-first-issue` labels.

**Q: How long until my PR is reviewed?**
A: Usually within 1-3 days. Automated checks run immediately.

**Q: Can I work on multiple things at once?**
A: Yes, but focus on one PR at a time for faster reviews.

**Q: What if my PR is rejected?**
A: Learn from feedback and try again. Rejection is part of learning!

**Q: How do I become a maintainer?**
A: Consistent, quality contributions over time. Show leadership and help others.
